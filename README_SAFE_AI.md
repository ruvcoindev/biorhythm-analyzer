# 🛡️ Безопасная работа с локальным AI (Ollama + Lazygit)

## 📚 Что настроено
Этот проект настроен для безопасной работы с **локальной** AI-моделью.
Все изменения отслеживаются, и ты всегда можешь откатиться к рабочей версии.

## ⚡ Быстрый старт

Перед тем как попросить локальную модель изменить код:

```bash
# 1. Создай точку восстановления
./scripts/git-safe-ai.sh checkpoint

# 2. Запроси изменения у локальной модели
ollama run deepseek-coder:6.7b "твой запрос"

# 3. Проверь изменения
git diff

# 4. Протестируй сборку
go build ./...

# 5. Если всё ок — коммить
git add -A && git commit -m "Описание"

# 6. Если сломалось — откат
./scripts/git-safe-ai.sh rollback 1 y

🔧 Инструменты

1. Lazygit

Горячие клавиши:
u — отменить последний коммит (сохраняя изменения)
d — удалить изменения
c — создать ветку от коммита
b — переключить ветку

2. Git Hooks

Перед каждым коммитом автоматически:
✅ Проверяется go build ./...
✅ Форматируется код go fmt
❌ Если сборка не проходит — коммит отменяется!

3. Скрипт git-safe-ai.sh

      Команда                                Описание

checkpoint                        Создать ветку-чекпоинт

backup                           Сохранить в stash

restore N                        Восстановить из stash

rollback                        Интерактивный откат

history                        Показать историю

📋 Рабочий процесс с локальным AI

Шаг 1: Подготовка

```bash

go build ./...
./scripts/git-safe-ai.sh checkpoint

Шаг 2: Запрос к модели

Всегда указывай:
Какую проблему решить
Что НЕЛЬЗЯ менять
Что протестировать после

Пример запроса для ollama run:

Проблема: функция X возвращает ошибку при условии Y.

ЗАПРЕЩЕНО:
- Менять функции A, B, C (они работают)
- Удалять тесты

РАЗРЕШЕНО:
- Добавить обработку ошибки в функцию X
- Добавить тест для граничного случая

Сначала покажи план, потом выполняй.
После изменений проверь: go build ./...

Шаг 3: Проверка

```bash

git diff          # что изменилось
go build ./...    # сборка
go test ./...     # тесты (если есть)


Шаг 4: Решение

✅ Всё работает → git add -A && git commit -m "..."
❌ Сломалось → ./scripts/git-safe-ai.sh rollback 1 y

🚨 Аварийное восстановление
Если локальная модель сломала код:

```bash

# Вариант 1: Быстрый откат через скрипт
./scripts/git-safe-ai.sh rollback 1 y

# Вариант 2: Жёсткий откат
git reset --hard HEAD

# Вариант 3: Переключение на чекпоинт
git branch | grep checkpoint
git checkout checkpoint_YYYYMMDD_HHMMSS

💡 Советы

Маленькие шаги: проси модель менять по 1 файлу за раз
Частые коммиты: после каждого успешного изменения — коммить
Явные инструкции: чётко указывай что можно/нельзя
Тестирование: всегда go build ./... после изменений
Резервные копии: перед крупным рефакторингом — отдельная ветка
Главное правило: Если сомневаешься — создай чекпоинт!
