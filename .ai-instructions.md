# 📋 Инструкция для локальной AI-модели (Ollama + DeepSeek-Coder)

## ⚠️ КРИТИЧЕСКИ ВАЖНЫЕ ПРАВИЛА

### 1. Перед любыми изменениями:
```bash
./scripts/git-safe-ai.sh checkpoint

ВСЕГДА создавай чекпоинт перед внесением изменений!

   2.Запрещено:

❌ Менять рабочий код без тестирования
❌ Удалять существующие функции, если они работают
❌ Вносить массовые изменения без согласования
❌ Игнорировать ошибки компиляции

    3.Разрешено:

✅ Минимальные изменения для исправления конкретной ошибки
✅ Предлагать альтернативное решение вместо правки текущего кода
✅ Сначала показать план, потом выполнять
✅ Тестировать каждое изменение перед следующим шагом

🔍 Процесс отладки

Когда обнаружена ошибка:

  1. СТОП! Не меняй код сразу
  2. Проанализируй и покажи:
    Что сломалось
    Где проблема
    Варианты решения
  3. Предложи план:

       План:
 1. Чекпоинт текущей версии
 2. Изменить файл X, строки Y-Z
 3. go build ./...
 4. Если ок → коммит, если нет → откат

  4.Только после подтверждения — выполнять

🛡️ Команды безопасности

       Команда                                       Описание

./scripts/git-safe-ai.sh checkpoint            Точка восстановления

./scripts/git-safe-ai.sh rollback 1 y          Быстрый откат

./scripts/git-safe-ai.sh history               Показать историю

git diff                                       Посмотреть изменения

git stash                                      Временно спрятать изменения


 💡 Лучшие практики

Маленькие шаги: одно изменение = один коммит
Тестирование: после каждого изменения go build ./...
Откат: если что-то пошло не так — сразу откатывайся
Документирование: коммить с понятными сообщениями

 🚨 Экстренный откат

```bash 

# Вариант 1: Жёсткий откат
git reset --hard HEAD

# Вариант 2: Через скрипт
